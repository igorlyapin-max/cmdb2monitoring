Общий принцип

                
    
                  
  • Браузер работает только с monitoring-ui-api; прямых подключений из браузера к CMDBuild, Zabbix или Kafka нет.
    • 
                  
  • Все адреса, учетные данные, Kafka topics, IdP/SAML2 и параметры чтения Events настраиваются во внешних конфигурационных файлах или через Settings.
    • 
                  
  • Всплывающие подсказки показываются при наведении или фокусе на элементе интерфейса.
    • 
                

              

              

                
Dashboard и Events

                
    
                  
  • Dashboard показывает доступность cmdbwebhooks2kafka, cmdbkafka2zabbix, zabbixrequests2api и самого BFF.
    • 
                  
  • Events читает последние сообщения из настроенных Kafka topics через backend-адаптер.
    • 
                  
  • Количество в Events означает "последние N сообщений"; по умолчанию выводятся 5 последних.
    • 
                

              

              

                
Rules

                
    
                  
  • Load загружает текущий JSON правил конвертации.
    • 
                  
  • Validate проверяет структуру правил на backend.
    • 
                  
  • Dry-run применяет правила к тестовому CMDBuild payload без сохранения.
    • 
                  
  • Upload сохраняет новый JSON правил на backend после проверки.
    • 
                

              

              

                
Mapping

                
    
                  
  • Страница показывает цепочку CMDBuild -> Conversion Rules -> Zabbix.
    • 
                  
  • Режим просмотра показывает выбранную цепочку и скрывает лишние атрибуты классов, другие rule-подблоки и несвязанные элементы списков Zabbix.
    • 
                  
  • Снять выделение доступно в режиме просмотра и возвращает Mapping к обычному обзору без подсветки цепочки.
    • 
                  
  • Режим редактирования меняет draft JSON текущей сессии: можно выбрать CMDBuild class/class attribute field, структуру конвертации, Zabbix object/payload, priority и regex.
    • 
                  
  • Действие "Удаление правила" показывает rules текущего draft JSON, удаляет отмеченные правила после подтверждения и оставляет классы/class attribute fields на месте.
    • 
                  
  • Undo и Redo работают с историей изменений текущей сессии, а Save file as сохраняет draft в отдельный JSON-файл без отправки на backend.
    • 
                  
  • Save file as дополнительно проверяет, что каждый настроенный класс имеет IP или DNS class attribute field, связанный с Zabbix interface rules или hostProfiles[].interfaces.
    • 
                  
  • Повторное нажатие на выбранный элемент снимает выделение.
    • 
                  
  • Interface address rules выбирают, чем заполнить Zabbix interface: ipAddress дает useip=1, dnsName дает useip=0.
    • 
                  
  • Host profiles описывают два режима: несколько interfaces[] внутри одного Zabbix host или несколько Zabbix hosts из одного CMDB object.
    • 
                  
  • Внутри host profile interface profile выбирает тип мониторинга agent/SNMP/IPMI/JMX, а valueField указывает, какой CMDB атрибут станет IP или DNS.
    • 
                  
  • Для Server webhook keys interface/interface2/profile/profile2 связаны с реальными CMDBuild attributes iLo/iLo2/mgmt/mgmt2 через cmdbAttribute; это нужно для Mapping и Body, но не добавляет старые alias в обработку payload.
    • 
                  
  • Для lookup выделяется только конкретная связка класса, lookup и значения, например Notebook.zabbixTag.tag1.
    • 
                  
  • Regex в правилах показывает, по каким class attribute fields выбираются группы, шаблоны, tags и расширенные Zabbix-объекты.
    • 
                

              

              

                
Validate rules mapping

                
    
                  
  • Страница не строит интерактивную цепочку, а подсвечивает только отсутствующие сущности.
    • 
                  
  • Красным отмечаются классы и атрибуты, отсутствующие в CMDBuild catalog, а также Zabbix-ссылки, которых нет в Zabbix catalog.
    • 
                  
  • Checkbox над отсутствующим элементом включает его в удаление из rules JSON.
    • 
                  
  • Delete спрашивает подтверждение, сохраняет предыдущую версию и исправляет выбранные ссылки в правилах.
    • 
                

              

              

                
Каталоги и Settings

                
    
                  
  • Zabbix Catalog загружает templates, host groups, template groups, tags и расширенные справочники Zabbix.
    • 
                  
  • CMDBuild Catalog загружает классы, атрибуты и lookup-значения.
    • 
                  
  • Settings сохраняет runtime-настройки подключений, Events Kafka browser и IdP/SAML2.
    • 
                  
  • Справочники источников лучше менять в CMDBuild/Zabbix, а правила конвертации менять в JSON rules.
